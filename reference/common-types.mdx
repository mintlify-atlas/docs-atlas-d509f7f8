title: "Common types and enums reference"
description: "Reference for all enums, type aliases, and shared structures in the shaiya library: Country, Family, Grow, Job, Sex, ItemType, UserTypes, and more."
---

The shaiya library provides common type definitions, enums, and structures shared across all Shaiya Core modules. These types live in `shaiya/include/common/` and cover game concepts like character class, faction, growth mode, item attributes, and packet-level bag identifiers. All enum values shown are taken directly from the header sources.

## Attribute.h

Elemental attributes used for attack and defense calculations. The type is `uint8_t`.

```cpp
enum struct Attribute : uint8_t
{
    None,
    Fire1,
    Water1,
    Earth1,
    Wind1,
    Fire2,
    Water2,
    Earth2,
    Wind2
};
```

The two tiers (`Fire1`/`Fire2`, etc.) correspond to the two lapisian elemental grades available in the game.

## Bag.h

Named constants for special bag identifiers used in inventory and warehouse routing. The enum uses `uint8_t` and also defines comparison operators against `int` for convenience.

```cpp
enum struct Bag : uint8_t
{
    Warehouse    = 100,
    Bank         = 200,
    Unknown      = 250,
    GuildWarehouse = 255,
    Skill        = 100,
    BasicAction  = 101
};
```

`Warehouse` and `Skill` share the value `100`; the correct interpretation depends on the context (item vs. skill routing). `Bank` refers to the personal bank/deposit box.

## Country.h

`Country` describes the faction alignment of mobs, NPCs, and users. `Faction` is a more granular breakdown of the same concept, covering individual races.

```cpp
// mobs, npcs, users
enum struct Country : uint8_t
{
    Light,
    Fury,
    Neutral
};

enum struct Faction : uint8_t
{
    Human,
    Elf,
    Light,
    Deatheater,
    Vail,
    Fury,
    Neutral
};
```

## CraftName.h

`CraftName` is a type alias for a fixed-length character array used to store an item's craft (composition) name:

```cpp
using CraftName = std::array<char, 21>;
```

It appears in item packet structures and in the item ability transfer effect to identify which craft data moves between source and target items.

## Family.h

`Family` identifies the playable race of a character. The type is `uint8_t`.

```cpp
enum struct Family : uint8_t
{
    Human,
    Elf,
    Deatheater,
    Vail
};
```

`Human` and `Elf` belong to the Light faction. `Deatheater` and `Vail` belong to the Fury faction. `BattleFieldMoveInfo.ini` maps one destination coordinate per family for battleground movement.

## Grow.h

`Grow` is the character growth mode selected at creation. The type is `uint8_t`.

```cpp
enum struct Grow : uint8_t
{
    Easy,
    Normal,
    Hard,
    Ultimate
};
```

Shaiya Core forces Ultimate Mode stat and skill-point growth paths for every mode via patched growth branches.

## ItemSlot.h

`ItemSlot` holds named slot indices for the equipment array. `ItemSlotCount` is the total number of equipment slots in the EP6.4 layout.

```cpp
constexpr int ItemSlotCount = 24;

struct ItemSlot
{
    enum
    {
        Helmet,
        UpperArmor,
        LowerArmor,
        Gloves,
        Boots,
        Weapon,
        Shield,
        Cloak,
        Necklace,
        Ring1,
        Ring2,
        Bracelet1,
        Bracelet2,
        Vehicle,
        Pet,
        Costume,
        Wings
    };
};
```

Slots `Vehicle` through `Wings` are the EP6.4 additions enabled by Shaiya Core's equipment hooks.

## PartyTypes.h

Three enums govern party loot distribution, party role, and raid/union role.

```cpp

};

enum struct UnionType : int32_t
{
    Default = -1,
    Boss,
    SubBoss,
    User
};
```

`UnionType` is used in the party boss map-position broadcast packet `0xB1C`.

