`CUser` is the central user/character object in `ps_game.exe`. Shaiya Core extends the standard `CUser` allocation from `0x62A0` to `0x62F4` to add custom runtime state fields. All offsets shown are from the base of the struct. The struct inherits from `CObjectConnection` and `UserInfo`, with the `CUser`-specific fields beginning at `0x121C`.

<Note>
Shaiya Core increases the CUser allocation size from `0x62A0` to `0x62F4`. If you use Cheat Engine scripts that access user memory, adjust them to account for the extended layout. The comment in `src/main.cpp` notes this explicitly.
</Note>

## Custom extended fields

Four fields are appended at the end of `CUser` after the base allocation ends at `0x62A0`:

```cpp
// custom
ItemQualityLv<24> itemQualityLvEx;  //0x62A0
ItemQuality<24>   itemQualityEx;    //0x62B8
SkillAbility70    skillAbility70;   //0x62E8
uint32_t          multiplyQuestExpRate; //0x62F0
```

`itemQualityLvEx` and `itemQualityEx` store extended item quality data for the EP6.4 equipment slot count of 24. `skillAbility70` tracks the state of skill ability type 70, which requires delayed cleanup after the skill stops. `multiplyQuestExpRate` holds the quest EXP multiplier sourced from skill ability type 87.

## SkillAbility70 struct

The `SkillAbility70` struct is packed and placed at offset `0x62E8`:

```cpp
#pragma pack(push, 1)
struct SkillAbility70 {
    uint16_t skillId;
    uint8_t  skillLv;
    bool     triggered;
    tick32_t keepTime;
};
#pragma pack(pop)
```

### UserMovePosType

`GateKeeper` and `Portal` share the same value (`1`). `TownMoveScroll` is a custom addition. Higher values use explicit hex assignments.

```cpp
enum struct UserMovePosType : int32_t
{
    Default,
    GateKeeper  = 1,
    Portal      = 1,
    SavePoint,
    RecallRune,
    PartyCall,
    PartyMove,
    MoveWar,         // itemId 100169 (EP5)
    TownMoveScroll,  // custom
    MoveMapId200    = 10,
    MoveChar        = 0xF904,
    MoveCharZone    = 0xF905,
    MoveParty       = 0xF90C,
    MoveTo          = 0xFA09,
    MoveZone        = 0xFA0A,
    MoveMap         = 0xFA11,
    MoveInsZone     = 0xFA12
};
```

## Size assertion

The header confirms the extended layout with a compile-time assertion. The commented-out original assertion (`0x62A0`) is preserved for reference:

```cpp
//static_assert(sizeof(CUser) == 0x62A0);
static_assert(sizeof(CUser) == 0x62F4);
```
